HTML5 新增 JavaScript API

2018-03-10 JavaScript HTML5, JavaScript 字数统计: 1k(字) 阅读时长: 4(分)

一、新增js选择器

  1. document.querySelector(“selector”)：根据CSS选择器语法选择单一DOM元素
  2. document.querySelectorAll(“selector”)：根据CSS选择器语法选择DOM元素列表
  3. document.getElementsByClassName(“className”)：根据元素class选择DOM元素列表
  4. obj.classList：获取元素class列表，返回类数组，提供如下方法：
    • add()：添加
    • remove()：删除
    • toggle()：切换

二、JSON

  1. JSON.parse() : 把严格模式的json形式字符串转成json
  2. JSON.stringify() : 把json转化成字符串
  3. 拷贝对象：b = JSON.parse( JSON.stringify(a) );

三、自定义属性

  1. html中设置：元素自定义属性用data-*建立
  2. js中获取：node.dataset 对象下保存所有元素自定义属性内容(DOMStringMap类型)，通过 node.dataset.* 获取，中间有多个连字符则用驼峰式命名获取

四、js延迟加载

(1). defer属性

作用：立即加载js文件，但延迟到 DOMContentLoaded 事件前顺序触发

DOMContentLoaded 事件：初始html文档完全加载和解析完毕之后触发，而无需等待样式表、图像和子框架的完成加载

(2). async属性

作用：指示浏览器在允许的情况下异步执行该脚本，不让页面等待脚本下载和执行，从而异步加载页面其他内容

(3). 动态创建script代码

五、历史管理

(1). 如何触发历史管理
  1. 浏览器手动前后页跳转
  2. hash 改变
  3. pushState 事件
(2). history 对象
属性：

history.length：浏览器历史信息总个数

历史记录跳转：
  1. history.back()：移动到上一个访问页面，等同于浏览器的后退键
  2. history.forward()：移动到下一个访问页面，等同于浏览器的后退键
  3. history.go(i)：接受一个整数作为参数，移动到该整数指定的页面，比如go(1)相当于forward()，go(-1)相当于back()。
历史信息管理：
  1. history.pushState(stateObj, title, URL)：添加和修改历史记录条目。参数为数据、标题、地址(可选)，地址必须是相同域名，不能跨域。数据大小不能超过640k
  2. history.replaceState(stateObj, title, URL)：修改当前历史记录信息，参数与 history.pushState() 相同
  3. 注意：以上两个方法仅改变历史信息，并不会触发hashchange事件
  4. history.state：保存当前激活 history 下储存数据的拷贝，注意数据仅通过 pushState() 或 replaceState() 方法添加
(3). window 事件
  1. window.onhashchange()：当 URL中 hash 改变时触发
  2. window.onpopstate()：激活状态的历史记录条目发生变化时触发，如浏览器前进后退按钮或调用 history.back()、history.forward()、history.go() 方法，history.pushState()或 history.replaceState()不会触发，事件处理函数中 event.state 为当前激活 history 下的 state 数据

六、FileReader 接口

(1). 属性

  1. FileReader.readyState：文件读取状态：EMPTY / LOADING / DONE

(2). 方法

  1. FileReader.readAsBinaryString(file)：将文件读取为二进制编码
  2. FileReader.readAsText( file, [encoding] )：将文件读取为文本
  3. FileReader.readAsDataURL(file)：将文件读取为data: URL 形式
  4. FileReader.readAsArrayBuffer()：将文件读取为 ArrayBuffer 形式
  5. FileReader.abort()：中断读取操作

(3). 事件

事件处理函数内 event.target.result 保存文件读取结果

  1. loadstart：开始
  2. progress：正在读取
  3. load：成功读取
  4. error：出错
  5. abort：中断
  6. loadend：读取完成，无论成功失败

七、window.postMessage

(1). 向另一窗口发送数据

  1. otherWndow：其他window对象的引用，可为 iframe.contentWindow 或使用 window.open() 返回的窗口对象
  2. message：传送的数据
  3. targetOrigin：允许接受信息的域名(要加协议)，"*"为不限制
  4. transfer：是一串和message 同时传递的 Transferable 对象. 这些对象的所有权将被转移给消息的接收方，而发送一方将不再保有所有权。

(2). 另一窗口接收数据

  1. event.data：从其他窗口传递的数据
  2. enevt.origin：发送方的域名，Chrome下使用 event.originalEvent.origin 获取
  3. event.source：对发送消息窗口的引用，可以做两个窗口间的双向通信
