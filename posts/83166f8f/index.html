Node.js Buffer and FileSystem

2018-07-05 Node.js JavaScript, Node.js 字数统计: 1.2k(字) 阅读时长: 4(分)

一、Buffer

1. 创建Buffer对象

  1. Buffer.from(array)：通过array创建一个新的Buffer

​ 例：Buffer.from([0x62, 0x75, 0x66, 0x66, 0x65, 0x72])

  1. Buffer.from(buffer)：将传入的buffer数据拷贝到一个新建的Buffer实例。
  2. Buffer.from(string[, encoding])：新建一个包含所给的JavaScript字符串string的Buffer，encoding参数指定string的字符编码。
  3. Buffer.alloc(size[, fill[, encoding]])，参数分别为长度，预填充的值( <string> | <Buffer> | <integer> )，字符编码( 默认: 'utf8’ )
  4. Buffer.allocUnsafe(size)：创建长度为size的buffer对象，内容未初始化，可能包含敏感数据，但性能更好

注：所有Buffer对象创建后长度固定，不能扩增

2. Buffer类方法

  1. Buffer.isEncoding(encoding)：判断是否支持某种编码格式
  2. Buffer.isBuffer(obj)：检测对象是否为Buffer类型
  3. Buffer.compare(buf1, buf2)：比较buf1和buf2，通常用于Buffer实例数组的排序，相当于调用buf1.compare(buf2)，返回-1/0/1
  4. Buffer.byteLength(string[, encoding])：返回一个字符串的实际字节长度，编码默认utf8

3. Buffer对象下的方法

  1. buf.write(string[, offset[, length]] [, encoding])：根据encoding的字符编码写入string到 buffer中的offset位置。 length 参数是写入的字节数。 如果buf没有足够的空间保存整个字符串，则只会写入 string 的一部分，只部分解码的字符不会被写入。
  2. buf.toString([encoding[, start[, end]]])：根据 encoding 指定的字符编码解码 buf 成一个字符串。 start 和 end 可传入用于只解码 buf 的一部分。
  3. buf.toJSON()；返回type=‘Buffer’，data为十进制buffer内容的数组的对象
  4. buf.slice([start[, end]])：返回一个指向相同原始内存的新建的Buffer切片(有改动老的也会改)，但做了偏移且通过 start 和 end 索引进行裁剪
  5. buf.copy(target[, targetStart[, sourceStart[, sourceEnd]]])：拷贝新的buffer对象

二、FileSystem

1. 打开

  1. fs.open(path, flags[, mode], (err, fd) => {} )：异步地打开文件，参数为文件路径、打开方式(读/写/追加)、文件模式
  2. fs.openSync(path, flags[, mode])：fs.open()的同步版本，返回一个表示文件描述符的整数(fd)

2. 监视

  1. fs.watch(filename[, options] [, (eventType, filename) => {} ])：监视filename的变化，filename可以是一个文件或一个目录(文件夹)，返回的对象是一个fs.FSWatcher
  2. fs.watchFile(filename[, options], (current, previous) => {} )：监视filename的变化，每次访问文件时调用回调函数，回掉中参数为两个fs.Stats对象

3. 读

  1. fs.read(fd, buffer, offset, length, position, (err, bytesRead, buffer) => {} )
    • fd：文件中读取数据
    • buffer：数据将被写入到的buffer
    • offset：buffer中开始写入的偏移量
    • length：指定要读取的字节数
    • position：从文件中开始读取的位置
    • 回掉
  2. fs.readSync(fd, buffer, offset, length, position)：fs.read()的同步版本
  3. fs.readFile(path[, options], (err, data) => {} )：异步地读取一个文件的全部内容，如果未指定字符编码，则返回原始的buffer，如果options是一个字符串(如utf8)，则它指定了字符编码
  4. fs.readFileSync(path[, options])：fs.readFile()的同步版本
  5. fs.stat(path[, options], (err, fs.stats) => {} )：读取文件信息，回掉函数中参数返回fs.stats
  6. fs.statSync(path[, options])：fs.stat()的同步版本，返回fs.stats

4. 写

  1. fs.write(fd, buffer[, offset[, length[, position]]], (err, bytesWritten, buffer) => {})
    • fd：指定的文件
    • buffer：写入的buffer数据
    • offset：buffer中写入的起始位置
    • length：写入的字节数
    • position：文件写入数据的位置的偏移量(起始位置)
    • 回掉
  2. fs.writeSync(fd, buffer[, offset[, length[, position]]])
  3. fs.write(fd, string[, position[, encoding]], (err, written, string) => {} )
  4. fs.writeSync(fd, string[, position[, encoding]])
  5. fs.writeFile(file, data[, options], (err) => {} )：异步地写入数据到文件，如果文件已经存在，则覆盖文件，data可以是字符串或buffer
  6. fs.writeFileSync(file, data[, options])：同步的fs.writeFile()
  7. fs.appendFile(path, data[, options], (err) => {} ):将数据写入文件尾部，如果文件不存在则创建文件，data可以是一个字符串或Buffer
  8. fs.appendFileSync(path, data[, options])：fs.appendFile()的同步方法

5. 删除

  1. fs.unlink(path, (err) => {} )：删除文件
  2. fs.unlinkSync(path)：fs.unlink()的同步版本，返回值为undefined

6. 重命名

  1. fs.rename(oldPath, newPath, (err) => {} )：重命名
  2. fs.renameSync(oldPath, newPath)：fs.rename()的同步版本，返回值为undefined

7. 关闭

  1. fs.close(fd, (err) => {} )：关闭一个打开的文件
  2. fs.closeSync(fd)：fs.close()的同步版本

8. 文件夹操作

  1. fs.mkdir(path[, mode], (err) => {} )：创建目录，完成回调只有一个可能的异常参数，mode 默认为0o777。
  2. fs.mkdirSync(path[, mode])：fs.mkdir()的同步版本，返回 undefined
  3. fs.rmdir(path, (err) => {} )：删除文件夹
  4. fs.rmdirSync(path)：fs.rmdir()的同步版本
  5. fs.readdir(path[, options], (err, files) => {} )：读取一个目录的内容，得到文件列表
  6. fs.readdirSync(path[, options])：fs.readdir()的同步版本，返回文件列表
